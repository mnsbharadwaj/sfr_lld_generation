# IP-XACT Excel + existing sfr.h to header-only lld.h generator

Phase 7.1 implements the customer requirement:

- `sfr.h` is generated by another tool and is treated as authoritative for offsets, masks, and shifts.
- Excel provides the IP address map, register fields, access type, and descriptions.
- The generator creates `lld.h` only.
- `lld.h` contains `static inline` function bodies, not only declarations.
- No AUTOGEN markers are used.
- Existing `lld.h` is patched by AST-style function/prototype indexing using function names and byte offsets.

## Install

```bash
pip install -r requirements.txt
```

## Windows PowerShell run

```powershell
$env:PYTHONPATH="src"; python -m ipxact_lld_gen.cli --excel sample/ipxact_sample.xlsx --sfr-header sample/sfr_new.h --existing-lld sample/old_lld.h --sfr-diff sample/sfr.diff --out out_patch
```

## CMD run

```bat
set PYTHONPATH=src
python -m ipxact_lld_gen.cli --excel sample/ipxact_sample.xlsx --sfr-header sample/sfr_new.h --existing-lld sample/old_lld.h --sfr-diff sample/sfr.diff --out out_patch
```

## Outputs

- `expected_lld.h`: full expected header-only LLD with static inline functions.
- `patched_lld.h`: existing `lld.h` patched without markers.
- `patch_report.json`: replaced/inserted/unchanged API list.
- `semantic_report.json`: derived semantics from Excel descriptions.
- `ast_declaration_index.json`: parsed declarations/functions from existing `lld.h`.

## Generated API behavior

If description is empty:

- setter/getter static inline functions are generated based on access type.

If description says W1C / write-one-to-clear:

- clear helper is generated.

If description says start/trigger/initiate:

- start helper is generated. If DONE/ERROR related fields are detected, the helper polls/checks them.
